Dépôts de Mods

OvGME Afterburner vous permet de configurer des Dépôts de mods distants pour le téléchargement ou la mise à jour de mods. Ce chapitre ne traite de la fonctionnalité que vu du coté client. Si vous comptez héberger et gérer un dépôt, veuillez consulter le chapitre Gestion de dépôt.

Les Dépôts de mods sont des serveur web distants qui fournissent une liste des mods à télécharger ou à mettre à jour. Ces serveurs doivent être OvGME Afterburner pour être reconnus durant la requête des Dépôts.

Note: les Dépôts sont fournis par des tiers-partis. OvGME Afterburner ne fournit aucun dépôt par défaut (voir Gestion de dépôt).

Configurer les Dépôts

Pour configurer les Dépôts distants pour la configuration actuel, allez dans le menu "Mods" sélectionnez "Repositories" puis "Configure...". Cela ouvrira une nouvelle boite de dialogue pour gérer les URL de Dépôts.
Note: La liste des Dépôts n'est pas globale, chaque configuration a sa propre liste de Dépôts. Si vous configurez les Dépôts pour une config, cette liste n'est valide que pour la config actuellement sélectionné.
Ajouter ou supprimer une URL
Pour ajouter une nouvelle URL de dépôt dans la liste, remplissez le champ de texte URL puis cliquez sur le bouton "Add". L'URL doit étre une URL valide vers un serveur HTTP. Voici quelques exemples de format d'URL valides:
"http://www.example.com/repos"
"www.example.com:8080/repository/dcs"
"http://192.168.0.1:80/mods"
Pour supprimer une URL de dépôt de la liste, sélectionnez l'URL désirés dans la liste et cliquez sur le bouton "Remove".
Activer ou désactiver une URL
Les Dépôts peuvent être activés ou désactivé pour la requête. Un dépôt désactivé ne sera pas interrogé durant l'interrogation des Dépôts, mais restera disponible dans la liste pour un usage future.
Pour activer ou désactiver une URL de dépôt, cochez ou d�cochez simplement la checkbox correspondant à l'URL dans la liste.

Interrogation des Dépôts

En interrogeant les dépots OvGME Afterburner va interroger chaque URL de dépôt activé pour créer la liste des mods disponibles pour le téléchargement ou la mise à jour. Pour interroger les Dépôts, allez dans le menu "Mods", sélectionnez "Repositories" puis "Query...". Cela vous ouvre une nouvelle boite de dialogue.

OvGME Afterburner va automatiquement démarrer le processus d'interrogation. Une fois l'interrogation terminée, et si elle réussie, des mods devraient apparaître dans la liste. Chaque entrée est identifié par une une icone qui décrit son statut relativement a votre liste de mods locaux pour cette configuration.
: Signifie que le mod est absent de votre dossier de mods local et peut être téléchargé.
: Signifie que le mod est déjà présent dans votre dossier de mods local et est a la même version.
: Signifie que le mod est déjà présent dans votre dossier de mods local mais a une version inférieure à celle disponible dans les Dépôts (vous pouvez upgrader ce mod).
: Signifie que le mod est déjà présent dans votre dossier de mods local mais a une version supérieure à celle disponible dans les Dépôts (si vous téléchargez ce mod, cela va downgrader votre mod local).
téléchargement des mods
Pour télécharger des mods depuis les Dépôts, vous pouvez simplement double-cliquer sur le mod désiré dans la liste. Vous pouvez également sélectionner plusieur mods, pui cliquer sur le bouton "Download selected". Finalement, vous pouvez synchroniser tous vos mods locaux en cliquant sur le bouton "Download all".

Le téléchargement des mods a plusieurs statu selon l'étape d'installation.
Pending... : Signifie que le mod est dans la liste de téléchargement et en attente de procédure.
Disabling old... : Signifie que le mod local actuel est en processus de désinstallation. En effet, un mod doit être désactivé avant d'être mis à jour.
Downloading... : Signife que le mod est en cours de téléchargement.
Il y'a aussi plusieurs statut d'erreur possibles:
Host not found : Signifie que la téléchargement a échoué car le nom d'hote n'a put être résolu.
Connection error : Signifie que la téléchargement a échoué à cause d'une erreur de connexion au serveur.
HTTP encode error : Signifie que la téléchargement a échoué à cause d'un encodage de transfert HTTP non supporté (voir ci-dessous pour les détails).
HTTP error ### : Signifie que la téléchargement a échoué à cause d'une erreur HTTP (généralement 404: page introuvable).
Connection lost : Signifie que la téléchargement a échoué à cause d'une connexion perdue.
I/O Open error : Signifie que la téléchargement a échoué à cause d'une erreur de création de fichier pour écriture sur le disque local.
I/O Write error : Signifie que la téléchargement a échoué à cause d'une erreur d'écriture de donnée dans le fichier sur le disque local.
