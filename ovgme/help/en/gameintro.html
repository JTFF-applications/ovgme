Introduction

OvGME allow to manage mods for several games, software or other custom locations through the main instance. A configuration entry defines the parameters for one working directory. A configuration works with three components: The root folder, the mods folder, and the backup folder.

Root folder

The root folder is where OvGME will install mods when you enable them. This can be a game or software installation folder, or any other folder where you want mods to apply some changes.

Mods folder

The mods folder is where OvGME will seek for available mods. This is where you must store your mods for this destination root folder (i.e. game, software, etc. )

Backup folder

The Backup folder is where OvGME will keeps all original files of a root folder (i.e. game, software, etc. ) which are modified by installed mods. By default, OvGME keeps backups data in its own internal structure (ProgramData\OvGME), but a custom backup folder can be choosen for each configuration.

Note: Each configuration root folder path must be unique, you cannot create several configuration for the same root folder.
