Mods FAQ

I have two identical mod title in the mods list, one is Directory type, the other is Zip Archive type, why ?

OvGME identifies mods by their name, the type is a secondary identification. So, if in the mods folder there is a Directory-Mod (sub-folder) named "My Mod v1.0" AND a valid mod archive named "My Mod v1.0.zip", OvGME will see both as valid mod to be installed. This is an ambiguous but not dangerous situation since you can't enable both versions: if you enable either one or the other, the duplicate Mod-Archive and Directory-Mod versions disappear to let only one Enabled version.

I have removed a mod from the mods folder, but it still in the list as Enabled ?

Fortunatly YES. Once a mod is enabled (installed), the backup data is stored in a safe location and OvGME keeps a look to the available data to restore. So the mod still visible in the list as an installed one and backup data sill available for restore. Once you disabled the mod, if the mod is no longer available in the mods folder, it will disappear from the mods list.

Where are stored backed files by default ?

If you don't have chosen a custom backup directory for the selected configuration, the backed files are stored in a "backups" named sub-folder in the dedicated configuration sub-folders within the OvGME application home directory indicated by your system, usually: "ProgramData\OvGME\<config Sub-folder>\backups".
