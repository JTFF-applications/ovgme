FAQ dépôts

OvGME prend-il en charge les serveur HTTPS ?

Non. OvGME ne prend en charge que le protocole HTTP standard.

Peut-on avoir plusieurs dépôts sur le même serveur (site web) ?

Oui. OvGME regarde seulement les fichiers XML à l'URL donnée et peu importe le serveur web, donc vous pouvez héberger autant de fichier XML que vous voulez au même endroit. Par exemple si vous voulez héberger des dépôts pour plusieurs jeux ou plusieurs listes de Mods, vous pouvez fournir un fichier XML pour chacun:
http://www.example.com/main (.xml)
http://www.example.com/updates (.xml)
http://www.example.com/special (.xml)
etc..

Est-ce que OvGME prend en charge les données XML générée dynamiquement (PHP / ASP pages) ?

Oui. OvGME peut receuillir les données XML depuis une page générée dynamiquement comme une page PHP ou ASP depuis la version 1.6, mais l'URL de dépôt fournit doit être explicite. Veuillez consulter le chapitre Principe du dépôt pour plus de détails.

Le serveur de dépôt doit-il également héberger les fichier zip de Mod-Archive ?

Non, comme vous pouvez le lire dans le chapitre Données XML de dépôt, les URL de mod(s) fournies dans l'XML sont totalement indépendantes de l'URL du dépôt. Le serveur de dépôt fournit uniquement le fichier "base de donnée" XML. Les fichier zip Mod-Archive peuvent être situés et téléchargeables à une toute autre adresse web.

OvGME prend-il en charge le téléchargement/mise à jour pour les mods qui sont pas des archives zip ?

Par pour le moment. OvGME va seulement regarder pour des fichier zip Mod-Archive. Durant le processus de dépôts (vérification des noms, comparaison de version, etc.) les Directory-Mod classiques sont tout simplement ignorés. Pour ainsi dire, le support des Directory-Mod classiques est plus ou moins destiné à devenir uniquement une fonctionnalité pour les créateurs de mods, pour aider au développement (puisqu'un dossier non compressé est plus facile à modifier qu'une archive zip).

Peut-on utiliser le mécanisme des dépôts dans un réseau local ?

Oui, OvGME agit comme un (très) simple client HTTP brute (navigateur web). Vous avez seulement besoin d'un serveur HTTP dans votre réseau local. Si votre réseau local n'a pas de service de nom de domaine (ce qui est souvent le cas), vous devez simplement entrer l'IP-URL du serveur en tant qu'URL de dépôt, par exemple: "http://192.168.0.5/repository/main"

Est-ce que le fichier zip disant à télécharger doit avoir exactement le même nom que celui du mod spécifié dans les données XML ?

Non. OvGME se fiche du nom du fichier à télécharger, une fois téléchargé, le fichier sera renommé selon le nom du mod spécifié avec l'extension ".zip". Par exemple, si dans les données XML, le nom spécifié du mod est "Mon Super Mod" et que l'URL de téléchargement est "http://www.ex.net/super-mod-v1_0.zip", le fichier téléchargé sera renommé en "Mon Super Mod.zip" et apparaitra en tant Mod-Archive zip nommé "Mon Super Mod" dans la liste des mods.
