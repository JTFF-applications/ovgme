Comparaison de snapshot

Pour comparer un snapshot existant pour la configuration actuelle, allez dans le menu "Config", sélectionnez "snapshot" puis "compare...". Une nouvelle boite de dialogue vous invitera a commencer le processus de comparaison. Le processus de comparaison peut prendre un long moment étant donné qu'il doit lire tous les fichiers pour vérifier le checksum xxHash. Vous pouvez annuler le processus tout moment.

Compte-rendu de comparaison

Durant le processus de comparaison, un compte rendu est généré dans la fenêtre du processus de comparaison. Ce compte-rendu liste toutes les différences de l'arborescence des fichiers avec le dernier snapshot. Cette liste compte trois colonnes: Type, Change et Path.

Type : identifie si l'entrée concerne un fichier ou un répertoire.
(f) : signifie Fichier.
(d) : signifie Répertoire.
Change : identifie quel type de changement l'entrée représente.
+ : signifie ajout, le fichier ou le dossier a été ajouté depuis le dernier snapshot.
- : signifie supprimé, le fichier ou le dossier a été supprimé depuis le dernier snapshot.
~ : signifie modifié, le fichier a été modifié depuis le dernier snapshot.
Path : identifie le chemin relatif du fichier ou du dossier concerné.
